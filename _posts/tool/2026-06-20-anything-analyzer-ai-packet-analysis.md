## 一句话

**Anything Analyzer** 是一个 AI 驱动的全场景抓包分析工具——把抓包和 AI 分析揉到一起，自动筛掉静态资源和噪声请求，拎出真正有价值的接口做深度分析。

> Mouseww/anything-analyzer ⭐ 3K | 🟦 TypeScript | MIT | 2026 年 4 月发布

## 为什么需要它？

做逆向或者抓包调试的人应该都熟悉这个场面：

Charles、mitmproxy、Fiddler 一开，列表瞬间刷满。几百条请求还算能忍，上千条请求堆在一起的时候，真正费时间的不是分析接口，而是先把"没用的"挑出去——图片、CSS、字体、埋点、统计、广告。。。这些噪声请求占了 80%+ 的列表，但每次都得手动忽略。

Anything Analyzer 的思路：**不是再做一个单纯抓包工具，而是把 AI 分析和抓包揉在一起。** 流量先抓进来，AI 先帮你过一遍，静态资源和噪声链路先筛掉，再把更有价值的接口拎出来做深挖。这一下就不是"人盯列表"，而是"人看结果"。

## 核心能力

### 全场景流量捕获

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| **内嵌浏览器** | 工具内直接操作页面、触发请求 | 日常 Web 调试 |
| **CDP 抓包** | 通过 Chrome DevTools Protocol 捕获 | 浏览器自动化场景 |
| **MITM 代理** | 中间人代理捕获 | 手机 App、桌面应用 |
| **CLI 命令行** | 终端命令捕获流量 | 脚本化/自动化 |
| **Python 脚本** | 通过 SDK 编程式捕获 | 集成到测试流程 |
| **移动端流量** | 配置代理后捕获手机 App 请求 | App 逆向分析 |

### AI 自动过滤

工具会先过滤无关请求（静态资源、埋点、统计等），再对关键请求做深度分析，最后给出分析报告。

对于那种接口链路长、调用关系绕、参数藏得深的场景，这种流程比手动一条条翻省时间太多。

### 深度分析报告

AI 会对选中的关键请求做：
- 参数语义理解（这个参数是干什么的）
- 接口用途推断（这个 API 是做什么的）
- 加解密/签名逻辑分析
- 请求链路的依赖关系图

## 技术架构

- **语言**：TypeScript
- **抓包引擎**：浏览器 CDP + MITM 代理双引擎
- **AI 分析**：集成 LLM 对请求和响应做语义分析
- **MCP Server 支持**：可作为 AI Agent 的工具，让 Agent 直接调用流量分析能力
- **安装包**：提供开箱即用的 Windows/macOS/Linux 安装包

## 适用场景

**接口逆向分析**——陌生 App 或者第一次接手的系统，AI 帮你过一遍快速定位关键接口，把前期摸索时间砍掉一截。

**安全测试**——捕获流量后 AI 自动分析数据泄露风险、未授权访问等安全隐患。

**调试排障**——海量请求中快速定位出问题的接口，不用手动一条条翻。

**AI Agent 集成**——通过 MCP Server 让 Agent 直接调用流量分析能力，自动分析接口逻辑。

## 总结

抓包本身早就不是最难的，难的是**在海量请求里尽快定位到值得看的那几条**。

Anything Analyzer 的方向戳中了这个痛点——用 AI 完成第一道过滤，让人只处理真正需要关注的请求。对于接口链路长、调用关系绕、参数藏得深的场景，这种流程比手动翻省时间太多。

当然，最后好不好用还是看 AI 过滤的准确率高不高，别把关键请求一起当噪声丢了。但从方向上来说，这是过去十几年抓包工具领域最有意义的创新之一。
