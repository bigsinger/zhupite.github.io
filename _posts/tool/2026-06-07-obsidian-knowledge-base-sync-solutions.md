## 前言

**同步**大概是使用 Obsidian 以来最让人头疼的问题之一。Obsidian 不像 Notion 那样自带云同步能力，它是一款基于本地 Markdown 文件的笔记工具，多设备之间的数据同步完全由用户自己解决。

本文系统梳理了 **6 种 Obsidian 同步方案**，覆盖从免费到付费、从简单到折腾的全光谱，你总能找到适合自己的那一款。

| 方案 | 费用 | 上手难度 | 平台支持 |
|------|------|---------|---------|
| Obsidian 官方同步 | $4-10/月 | ⭐ 极简 | 全平台 |
| iCloud | 免费（需苹果设备） | ⭐ 极简 | Apple 生态 |
| GitHub/Git | 免费 | ⭐⭐⭐ 较高 | 全平台 |
| 百度网盘同步空间 | ~178元/年 | ⭐ 简单 | Windows + 移动端 |
| Syncthing | 免费 | ⭐⭐ 中等 | 全平台 |
| 自建 NAS | 硬件成本 + 电费 | ⭐⭐⭐ 较高 | 全平台 |

## 一、Obsidian 官方同步 — 省心但贵

### 核心特性

Obsidian 官方提供了名为 **Obsidian Sync** 的付费插件，是唯一一种「开箱即用、无需任何配置」的同步方案。

| 项目 | 说明 |
|------|------|
| 价格 | $4/月（基础版10GB）、$10/月（100GB） |
| 加密 | 端到端加密，服务端不可读 |
| 版本历史 | 支持（可恢复指定版本） |
| 冲突处理 | 自动合并，冲突文件备份 |
| 选择性同步 | 可按文件夹/文件类型设置黑白名单 |

### 优点

- **零配置**：启用插件 → 登录 → 选择仓库 → 完成
- **端到端加密**：隐私性最优，Obsidian 官方也看不到你的笔记内容
- **版本历史**：误删或误改可以一键恢复
- **冲突处理**：同时编辑多设备后的自动合并机制足够成熟
- **选择性同步**：可以排除 `.obsidian/workspace` 等临时文件，节省同步空间

### 缺点

- **价格不低**：$4/月 ≈ 28元/月，一年 336 元，对非重度用户偏贵
- **国内访问延迟**：Obsidian 服务器在海外，国内部分网络环境下同步速度不稳定
- **不解决备份问题**：同步 ≠ 备份，误操作删除全库仍然需要额外的备份方案
- **无法自定义存储位置**：所有数据必须走 Obsidian 的云端

### 适合谁

- 愿意为「省心」付费的用户
- 有多设备同步需求但不折腾技术的用户
- 对笔记隐私有高要求的用户（端到端加密是刚需）

## 二、iCloud — 苹果生态的最佳选择

### 核心特性

利用 iCloud Drive 存储 Obsidian 仓库文件，在 iPhone、iPad、Mac 之间实现无缝同步。这是苹果全家桶用户最「无感」的方案。

### 配置方法

1. 将 Obsidian 仓库文件夹放在 **iCloud Drive/Obsidian/** 目录下
2. 在其他苹果设备上打开 Obsidian → 选择「打开本地仓库」→ 选择 iCloud 中的对应文件夹
3. 等待 iCloud 自动同步即可

### 优点

- **免费**：iCloud 有 5GB 免费空间，纯文本笔记很难用超
- **无缝集成**：苹果设备间同步几乎零延迟
- **无需额外注册**：使用已有的 Apple ID

### 缺点

- **仅限苹果生态**：Windows 和 Android 用户无法使用（虽然 iCloud for Windows 可用，但体验远不如苹果设备）
- **文件冲突**：iCloud 的冲突处理不如 Obsidian Sync 智能，偶尔会出现文件副本（`filename 2.md`）
- **大文件同步慢**：如果笔记中包含大量图片或附件，iCloud 的同步速度不如专用方案
- **iCloud 同步延迟**：快速切换设备编辑时可能出现版本冲突

### 适合谁

- 苹果全家桶用户（Mac + iPhone/iPad）
- 不跨 Windows/Android 设备使用的用户
- 希望零成本解决同步问题的用户

## 三、GitHub/Git — 免费但门槛高

### 核心特性

将 Obsidian 仓库作为一个 Git 仓库托管到 GitHub（或其他 Git 平台），通过 pull/push 实现多设备同步。这是开发者最熟悉的方式。

### 配置步骤

#### 1. 初始化 Git 仓库

```bash
# 在 Obsidian 仓库目录初始化 Git
cd ~/Documents/Obsidian
git init
git add .
git commit -m "初始化 Obsidian 仓库"

# 关联远程仓库（先在 GitHub 上创建一个私有仓库）
git remote add origin git@github.com:yourname/obsidian-vault.git
git push -u origin main
```

#### 2. 其他设备克隆

```bash
# 在其他电脑上
git clone git@github.com:yourname/obsidian-vault.git
```

#### 3. 日常同步流程

```bash
# 开始工作前
git pull

# 工作结束后
git add .
git commit -m "更新笔记"
git push
```

#### 4. 进阶：使用 Obsidian Git 插件

安装社区插件 **Obsidian Git** 后，可以自动执行上述操作，设定一个定时（如每 10 分钟）自动 commit + push，减少手动操作。

### 优点

- **完全免费**：GitHub 私有仓库不收费
- **版本控制**：完整的 Git 历史，可以随时回退到任意历史版本
- **全平台支持**：只要有 Git 和 Obsidian 就能用
- **不依赖 Obsidian 官方**：数据完全自主可控

### 缺点

- **需要科学上网**：国内访问 GitHub 不稳定，push/pull 经常失败，这是最致命的问题
- **操作繁琐**：需要手动或定时执行同步操作，做不到实时同步
- **冲突处理不友好**：如果在两个设备上同时编辑同一个文件，Git 合并冲突对非技术用户很不友好
- **忽略文件配置**：需要正确配置 `.gitignore` 排除 Obsidian 的临时文件和工作区状态

#### `.gitignore` 推荐配置

```gitignore
# Obsidian 工作区状态
.obsidian/workspace
.obsidian/workspace.json
.obsidian/cache/

# 系统文件
.DS_Store
Thumbs.db

# 大文件（可以用 Git LFS 替代）
*.psd
*.zip
```

### 适合谁

- 有 Git 使用经验的开发者
- 能稳定访问 GitHub 的用户
- 不介意手动同步步操作的用户

## 四、百度网盘同步空间 — 国内用户的实用之选

### 核心特性

这是引子文章原作者最终选择的方案。百度网盘的「同步空间」功能（会员功能）可以在本地创建一个文件夹，自动与云端保持同步。把 Obsidian 仓库放在这个同步空间里，就能实现多设备间自动同步。

### 配置步骤

1. 开通百度网盘会员（超级会员约 178 元/年）
2. 打开百度网盘 PC 端 → 找到「同步空间」功能
3. 创建一个同步空间（或使用已有的）
4. 将 Obsidian 仓库文件夹**移动到**同步空间目录下
5. 在其他电脑上登录同一百度网盘账号 → 打开同步空间 → Obsidian 中打开相同仓库
6. 百度网盘会自动检测文件变化并同步

### 优点

- **便宜**：178元/年 ≈ 15元/月，比 Obsidian 官方同步便宜一半
- **国内网络友好**：百度网盘国内服务器，同步速度快且稳定
- **自动同步**：文件变化后自动同步，无需手动操作
- **跨平台**：支持 Windows、macOS、Android、iOS
- **额外容量**：同步空间本身有 10GB 起步的存储空间

### 缺点

- **需要会员**：百度网盘的同步空间功能需要超级会员才能使用
- **安全隐患**：数据存储在百度服务器，理论上百度可以访问你的笔记内容（无端到端加密）
- **文件锁冲突**：如果两台电脑同时修改同一个文件，可能出现同步冲突（百度网盘的冲突处理不如 Obsidian Sync）
- **同步延迟**：不是实时同步，有几十秒到几分钟的延迟
- **大文件限制**：同步空间对单个文件大小有限制

### 适合谁

- 国内用户，没有稳定科学上网环境
- 跨多个 Windows 电脑使用 Obsidian
- 对笔记隐私要求不高（不涉及敏感信息）
- 预算有限，不想花 $4/月

## 五、Syncthing — 免费开源的点对点同步

### 核心特性

Syncthing 是一款开源的点对点文件同步工具。它不依赖任何云服务器，两台设备之间直接同步，数据不经过第三方。

### 配置步骤

1. 在所有需要同步的设备上安装 Syncthing
   - Windows：下载 GUI 安装包
   - macOS：`brew install syncthing`
   - Android：从 F-Droid 或 Google Play 安装 Syncthing-Fork
   - Linux：`sudo apt install syncthing`
2. 运行 Syncthing，Web 管理界面默认在 `localhost:8384`
3. 添加设备：在设备 A 上点击「添加远程设备」→ 输入设备 B 的设备 ID
4. 在设备 B 上确认连接
5. 添加文件夹：将 Obsidian 仓库文件夹添加为同步文件夹
6. 配置文件夹共享给其他设备

### 优点

- **完全免费**：没有任何隐藏费用
- **隐私安全**：端到端加密的 P2P 传输，数据不经过任何服务器
- **跨平台**：Windows、macOS、Linux、Android 均支持
- **实时同步**：文件变化后几乎立即同步
- **版本控制**：内置文件版本历史（可配置保留策略）

### 缺点

- **需要两台设备同时在线**：如果设备 A 没开机，设备 B 就收不到同步
- **配置略复杂**：需要理解设备 ID、文件夹 ID 等概念
- **缺少 iOS 官方支持**：iOS 端只能通过第三方客户端或间接方案
- **内网穿透**：如果两台设备在不同网络下，需要中继服务器或配置 NAT 穿透（Syncthing 自带中继，但不保证速度）
- **大文件同步效率**：不如专业云存储方案

### 适合谁

- 注重隐私的用户
- 有多台电脑且在同一个局域网内使用
- 不排斥一些技术配置的用户
- 不希望数据上传到任何云服务的用户

## 六、自建 NAS+同步 — 技术玩家的终极方案

### 核心特性

使用群晖、威联通等 NAS 设备搭建私有云盘，通过 Synology Drive、Nextcloud 或 Seafile 等工具同步 Obsidian 仓库。这是最折腾但也是最可控的方案。

### 方案对比

| 同步方式 | 配置复杂度 | 实时性 | 客户端 | 备注 |
|---------|-----------|-------|-------|------|
| **Synology Drive** | ⭐⭐ | 实时 | Windows/macOS/Android/iOS | 群晖用户首选，体验接近 Dropbox |
| **Nextcloud** | ⭐⭐⭐⭐ | 实时 | 全平台 | 开源免费，需 Docker 或虚拟机 |
| **Seafile** | ⭐⭐⭐ | 近实时 | 全平台 | 专业级同步，性能优秀 |
| **WebDAV** | ⭐⭐ | 手动/定时 | 需第三方客户端 | 通用协议，移动端可选 |

### 以 Synology Drive 为例的配置

1. 在群晖 NAS 上安装 **Synology Drive Server** 套件
2. 在电脑上安装 **Synology Drive Client** 桌面应用
3. 选择一个本地文件夹作为同步目的地
4. 在 Obsidian 中打开该文件夹作为仓库
5. NAS 会自动同步各设备的文件变更

### 优点

- **数据完全自主**：所有数据存储在自己的 NAS 上，不受云服务商限制
- **全平台覆盖**：NAS 客户端覆盖所有主流平台
- **接近实时同步**：局域网的同步速度非常快
- **版本历史**：NAS 系统通常自带文件版本管理
- **不仅是同步**：NAS 还可以跑备份、自动归档、图片管理等多种功能

### 缺点

- **硬件成本高**：入门级 NAS 千元左右，加上硬盘一般 2000 起步
- **配置复杂**：需要了解网络配置、端口转发等知识
- **外网访问**：需要配置 DDNS 或内网穿透才能在外网同步
- **维护成本**：NAS 需要定期维护、更新固件、监控硬盘健康
- **电费**：NAS 需要 7x24 小时开机

### 适合谁

- 已经拥有或计划购买 NAS 的用户
- 对数据主权有极致要求的用户
- 有多个设备且需要高速局域网同步的用户
- 愿意投入时间和金钱换取完全可控的方案

## 八、如何选择适合自己的方案

### 决策树

```
你的预算如何？
├─ 预算充足（愿付≥$4/月）
│  └─ 想要省心 → **Obsidian 官方同步**
│
├─ 预算有限（不愿付钱/少花钱）
│  ├─ 苹果全家桶 → **iCloud**
│  ├─ 有 NAS → **Synology Drive / 自建 NAS**
│  ├─ 有技术背景 + 能科学上网 → **GitHub + Obsidian Git 插件**
│  ├─ 主打国内 Windows 电脑 → **百度网盘同步空间**
│  └─ 注重隐私 + 有技术能力 → **Syncthing**
│
└─ 实在不愿付费
   ├─ 跨平台 → **Syncthing**
   └─ 仅苹果设备 → **iCloud**
```

### 我的推荐

| 用户类型 | 推荐方案 | 备选方案 |
|---------|---------|---------|
| 普通用户、不愿折腾 | Obsidian 官方同步 | 百度网盘同步空间 |
| 苹果全家桶 | iCloud | Obsidian 官方同步 |
| 开发者、能科学上网 | GitHub + Obsidian Git | Syncthing |
| 国内 Windows 用户 | 百度网盘同步空间 | Syncthing |
| 隐私敏感型 | Syncthing | 自建 NAS |
| 老司机、已有 NAS | Synology Drive | Nextcloud |
