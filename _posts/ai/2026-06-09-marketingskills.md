## 一句话介绍

**Marketing Skills** 是一个面向 AI 编程助手的营销技能集合，覆盖 **CRO（转化率优化）**、**文案撰写**、**SEO**、**分析**、**增长工程**等核心营销领域。装上它，你的 AI 编程助手就不再只是写代码的工具，而是一个具备专业营销能力的技术合伙人。

项目由 Corey Haines 开发，目前 **32,518 ⭐**。

## 为什么需要这个

当代 SaaS 创始人往往是技术出身——能写代码、能搭产品，但不一定擅长营销。Marketing Skills 就是为了填补这个空白：**把专业营销人员的知识和流程，装入 AI 编程助手中**。

你不是在问 AI「怎么做营销」，而是告诉它你有一个营销任务，它自动调用对应的营销框架和最佳实践来处理。

## 技能组成

Marketing Skills 包含了多个独立的技能文件，每个聚焦一个营销领域：

| 技能 | 领域 | 核心能力 |
|------|------|---------|
| **product-marketing** | 基础 | 理解产品、受众、定位（所有其他技能的前提） |
| **cro** | 转化率优化 | A/B 测试设计、漏斗分析、转化路径优化 |
| **copywriting** | 文案撰写 | 着陆页文案、邮件营销、价值主张提炼 |
| **seo** | 搜索引擎优化 | 关键词研究、内容策略、技术 SEO 审计 |
| **analytics** | 数据分析 | 营销归因、指标设定、数据驱动决策 |
| **growth** | 增长工程 | 病毒传播、PLG 策略、增长实验 |
| **social** | 社交媒体 | 内容日历、平台策略、投放优化 |

## 技能如何协同工作

Marketing Skills 的技能之间有层级关系——所有技能在运行前会先读取 `product-marketing` 技能，理解你的产品、受众和定位，然后再执行具体任务。

```
产品营销（基础）
    ├── CRO（理解产品后才做转化优化）
    ├── 文案（理解产品后才写文案）
    ├── SEO（理解产品后才做关键词研究）
    ├── 分析（理解产品后才设定指标）
    └── 增长（理解产品后才设计增长策略）
```

这种设计避免了「不知道你在做什么就开始做」的常见问题。

## 快速上手

在支持 Agent Skills 的 AI 编程助手中添加 Marketing Skills：

```bash
# 克隆仓库
git clone https://github.com/coreyhaines31/marketingskills.git

# 或添加到你的项目中
```

使用示例：

```
# 分析这个着陆页的转化率问题
# 为我的 SaaS 产品撰写 5 个 A/B 测试方案
# 做一份竞争对手的 SEO 关键词差距分析
# 设计一个 PLG 增长实验
```

Agent 会自动调用对应的营销技能来处理这些请求。

## 适用人群

| 人群 | 价值 |
|------|------|
| **技术创始人** | 没有营销团队时自己做增长 |
| **独立开发者** | 产品有了，但不知道怎么推广 |
| **营销人员** | 用 AI 提升执行效率 |
| **增长团队** | 将最佳实践标准化为可重复流程 |

## 优劣势

| 优势 | 不足 |
|------|------|
| 专业营销框架即开即用 | 需要结合具体行业知识调整 |
| 技能间协同工作 | 不是自动化执行，而是知识辅助 |
| 覆盖营销全链路 | 需要 AI 编程助手环境 |
| 开源、社区贡献 | 部分技能深度取决于作者经验 |

## 深度思考

Marketing Skills 代表了一种趋势：**专业知识的民主化**。不在营销领域有多年经验的人，也可以通过这些技能获得「营销第一性原理」级别的建议。

这不是替代营销人员，而是让非营销背景的人（尤其是技术创始人）也能做出有营销专业度的决策。
