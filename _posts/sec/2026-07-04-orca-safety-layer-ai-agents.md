14|## 产品定位：Agent 操作的"安全中间件"
15|
16|Orca 的安全层解决方案面向自主 AI Agent 的**运行时防护**需求，不要求修改 Agent 自身的代码或架构。其核心定位是一个介于 Agent 运行环境与外部系统之间的安全中间件层：
17|
18|- **非侵入式**：对 Agent 无感，Agent 本身不需要做适配
19|- **运行时生效**：在 Agent 执行操作的过程中实时介入，而非仅在部署前做静态检查
20|- **可独立升级**：安全策略更新不影响 Agent 版本迭代
21|
22|## 三大核心功能
23|
24|| 功能 | 说明 |
25||------|------|
26|| 行为监控 | 实时记录 Agent 的工具调用、API 请求、文件操作等行为，建立正常行为基线 |
27|| 权限控制 | 基于最小权限原则，限制 Agent 在运行时可执行的操作范围和资源访问 |
28|| 异常检测 | 识别偏离基线的可疑行为，在 Agent 执行过程中实时阻断 |
29|
30|这套功能覆盖了 Agent 运行时安全的核心需求：**知道 Agent 在做什么、限制它能做什么、发现异常时及时阻断**。

32|## 市场背景与意义
33|
34|Orca 安全层的推出有几个值得关注的行业信号：
35|
36|**1. Agent 安全正从静态配置走向运行时防护**
37|
38|早期的 Agent 安全焦点在提示词注入、工具权限配置等静态层面。Orca 的运行时方案说明行业已意识到：Agent 在运行过程中可能产生动态威胁（如工具链滥用、权限提升、数据越权），这些无法通过事前检查完全覆盖。
39|
40|**2. "安全中间件"模式正在成型**
41|
42|不修改 Agent 本身，而是以中间件方式插入安全层——这一模式与 Web 应用中的 WAF（Web 应用防火墙）逻辑类似。如果 Agent 安全中间件成为行业标准，将催生独立的安全产品市场，而非仅依赖 Agent 框架自带的防护能力。
43|
44|**3. Agent 运营者将成为安全决策的关键角色**
45|
46|运行时安全层的决策权在 Agent 运营者手中（而非 Agent 开发者），这意味着安全策略的配置、调优和响应流程将越来越多地落在运营团队的肩上。
