## 四大能力域

### 1. Agent 行为监控（Agent Behavior Monitoring）

这一方向的工具负责观察 AI Agent 在执行过程中的行为序列，包括：工具调用链、数据访问模式、权限使用情况和异常偏离检测。与传统 EDR 不同，Agent 行为监控需要理解「意图—动作—结果」之间的语义关系，而非仅仅匹配签名或阈值。

代表性场景：
- 检测 Agent 是否执行了非授权的工具调用
- 识别 Agent 陷入对抗性循环或重复失败模式
- 追踪 Agent 与多个外部系统的交互路径

### 2. MCP 流量检测（MCP Traffic Detection）

随着 Model Context Protocol（MCP）成为 AI Agent 与工具/数据源交互的主流协议，MCP 流量本身成为新的攻击面。MCP 流量检测工具专注于：

- 识别 MCP 连接中的异常请求模式（如批量读取敏感数据）
- 检测 MCP Server 端返回的恶意或篡改内容
- 分析 MCP 会话中的上下文注入尝试
- 监控 MCP 工具的权限变更与未授权注册

这标志着安全检测从「HTTP API 层」下沉到了「Agent 协议层」。

### 3. Prompt 注入防御（Prompt Injection Defense）

Prompt 注入从概念验证阶段进入规模化防御阶段。2026 年的防御工具已具备以下能力：

- **输入过滤**：在用户输入到达 LLM 之前检测注入特征
- **输出验证**：检查 LLM 输出是否包含不应泄露的内部信息或意外指令
- **上下文边界管理**：区分系统指令、工具返回与用户输入的语义边界
- **对抗测试**：持续对 Agent 应用进行自动化 Prompt 注入测试

### 4. Agent 供应链验证（Agent Supply Chain Verification）

Agent 应用的供应链比传统软件更复杂：除了代码依赖，还有模型权重、Prompt 模板、工具定义和 RAG 数据源。供应链验证工具覆盖：

- MCP Server 清单与来源校验
- Prompt 模板的篡改检测与版本追溯
- 模型权重的数字签名验证
- 第三方工具和插件的安全评估

## 市场信号：从点工具到平台整合

最值得关注的趋势是整合信号的出现。2025 年底到 2026 年上半年，我们看到：

1. **传统安全厂商入场**：已有多个大型安全厂商通过收购或自建进入 AI 应用安全领域，将上述能力集成到已有的 SASE/SIEM 平台中
2. **初创公司分化与整合**：2024-2025 年涌现的 AI 安全初创公司开始分化——少数获得大额融资独立发展，更多被平台厂商收购
3. **MCP 安全成为新赛道**：伴随 MCP 协议的普及，专门针对 MCP 的安全工具成为独立增长点
4. **CISO 面临选型挑战**：当安全厂商都在宣传「AI 安全」时，CISO 需要区分真正的运行时防护能力和传统的策略管理能力

## CISO 视角的评估框架

面对整合期的 AI 应用安全工具，建议从以下维度评估：

| 评估维度 | 核心问题 |
|---------|---------|
| **运行时深度** | 工具是否真正嵌入 Agent 执行流程，还是仅在边界做策略检查？ |
| **协议适配** | 是否支持 MCP 等 Agent 通信协议的流量级检测？ |
| **供应链覆盖** | 是否覆盖模型、Prompt、工具、MCP Server 四类资产的验证？ |
| **集成成本** | 与现有 SIEM/SOAR/SASE 的集成难度和数据管道对接成本？ |
| **对抗能力** | 是否具备对抗性测试能力，还是只做静态规则匹配？ |
