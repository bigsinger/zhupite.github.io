# 概述

2026 年 6 月 3 日，OpenAI 对 Codex 进行了一次重大改版。这次更新标志着 Codex 从「程序员的编码助手」向「白领全能办公工具」的战略转型。

# 核心数据：500 万周活，6 倍增长

Codex 周活跃用户突破 **500 万**，相比 2 月桌面版上线以来增长了 6 倍。更值得关注的是结构变化：

- **非开发者用户**（分析师、营销、运营、设计师、研究员、投资人、金融人士）占比已达 **20%**
- 非开发者增速是开发者的 **3 倍以上**
- 数据分析任务周环比增长 **110%**
- 研究类任务增长 **37%**
- 知识产出物（报告、备忘录、合同、PPT、表格）增长 **36%**
- **60% 以上**用户一天内同时运行多个 Codex 任务（4 月中旬这个比例还不到一半）

# 六大角色插件：给 Codex 装上职业技能

这次的插件设计思路不是通用能力往每个人身上套，而是按角色定制。每个插件打包了一个职业常用的软件，用一句话下指令，Codex 在背后帮你跨软件完成整个流程。

| 插件 | 对接工具 | 典型场景 |
| --- | --- | --- |
| **数据分析** | Snowflake、Tableau 等 | 「上季度哪个区域的收入下降了，为什么」→ 查数据、做分析、生成图表 |
| **创意制作** | Figma、Canva 等设计/图库平台 | 给一份创意简报，直接出一组广告素材变体 |
| **销售** | Salesforce、HubSpot 等 CRM/沟通工具 | 找高优先级客户、准备会议材料、跟进邮件 |
| **产品设计** | Figma、Canva | 把静态线框图变成可点击原型，审计用户流程 |
| **公共股权投资** | FactSet、S&P、PitchBook 等金融数据终端 | 盈利分析、公司对比、投资逻辑验证 |
| **投行** | 尽调/研究工具 | 把研究和尽调材料转化成客户就绪的 Pitch 文档，做可比公司分析 |

后续还将推出企业财务、私募、营销策略、咨询、法律等插件，最终目标是开放生态，让合作伙伴自己开发插件上架。

# Sites：你说一句话，它给你一个网站

Sites 是本次更新中最有想象力的功能，面向 Business 和 Enterprise 用户预览开放。

**核心能力**：以前 Codex 帮你干完活，给的是一个文件；现在能直接生成一个交互式网页应用，通过 URL 分享给工作区内的人。

**典型场景**：

- 财务负责人把一个 Excel 模型变成在线场景规划器，管理层直接在网页上调参数对比假设
- 产品发布时把所有材料汇总成一个 Hub 页面
- 客户评审前生成包含产品更新、待解决问题、使用趋势的互动页面

VentureBeat 指出：这直接威胁 Tableau、Power BI 这类 BI 工具。过去从「想要一个交互应用」到「真的有一个」需要几周的开发周期，现在压缩到几分钟的对话。

# Annotations：圈哪改哪

这个功能解决「初稿之后怎么改」的问题。

以前只有程序员用来改代码，现在扩展到文档、表格和幻灯片：

- 投资分析中某段数据不确定 → 圈出来问「这个数据出处是哪里」
- 幻灯片上图表标签看不清 → 标注一下换更清晰的写法
- Codex 做的网页导航栏字体不喜欢 → 选中导航栏让它换一个

关键是**只修改选中部分，其他一字不动**。

# 战略方向：Codex 与 ChatGPT 将合并

在本次更新发布前，Codex 负责人 Tibo 发了一条半开玩笑的推文：「我们是不是该把 Codex 改名叫 ChadGPT？」

实际上，Tibo 的个人简介已写的是「Codex & ChatGPT」，两个产品在 OpenAI 内部已是同一个团队在管。大概率走向是：Codex 会并进 ChatGPT，或者反过来说，ChatGPT 会变成 Codex。聊天只是入口，干活才是正事。

OpenAI 想把产品从「你问我答」变成「你说一句，我替你做完」。

# 行业影响：谁该紧张

最该关注这件事的可能不是微软和 Google，而是一批 SaaS 公司。

**短期护城河仍在的**：Salesforce、Snowflake 这类坐在数据和行业 know-how 上的厂商，合规、深度集成、行业知识不是装个插件就能替代的。

**真正该紧张的**：核心卖点是「界面好用」的中间层产品——BI 看板、项目管理、报告生成器。

原因很简单：过去用一个 SaaS 工具，门槛是学会操作界面。现在一个市场总监跟 Codex 说「把上个月各渠道的 ROI 拉个表给我」，Codex 直接连 Snowflake 抓数据，生成可交互的 Sites 页面，发个链接老板就能看。

当用户不再需要学工具的操作方式，只需要说清楚自己要什么，「界面好用」就不再是壁垒了。

真正的壁垒会往两头集中：
- 要么是数据和合规的源头，谁都绕不开你
- 要么是能理解用户意图、调度所有工具的 Agent 层

卡在中间靠界面吃饭的，会最先感受到压力。
